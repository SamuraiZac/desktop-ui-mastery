# Imran Chaudhri - Attention as the Material

Twenty-plus years in Apple's Human Interface team (interface work spanning iPhone, iPad, Mac, Watch), later co-founder of Humane and the Ai Pin. Chaudhri's declared material is human attention: the interfaces he championed at Apple earned it gracefully, and his Humane venture, which tried to abolish the screen entirely, failed instructively. Both halves belong in this library.

## Core principles, translated to desktop UI

1. **Attention is the material you are shaping.** Every design decision either spends, respects, or returns the user's attention. Frame reviews this way: what does this screen cost in attention, and what does it pay back?
2. **Technology should recede as understanding grows.** The interface a user needs in hour one is heavier than what they need in year two. Design the experienced state deliberately: progressive chrome reduction, learned shortcuts, muscle-memory layouts that never reorganize themselves.
3. **Calm by default, vivid by exception.** From the Apple lineage he shaped: notifications, badges, and alerts are a debt against trust. An event earns interruption only if a reasonable user would want to be interrupted; everything else accumulates quietly in a place the user visits by choice.
4. **Invisible until summoned.** Humane's core intuition (computing present but not looming) is right even though the Pin's execution failed. Desktop version: ambient status surfaces, global summon (hotkey, palette), and a product that is excellent at being closed.
5. **The Humane lesson: never trade capability for ideology.** The Pin removed the screen before its replacement (voice, projection) matched the screen's bandwidth. Radical interface subtraction must beat the incumbent on the user's actual tasks, not merely on philosophy. Test the radical version against the boring version honestly, and let the boring version win when it wins.

## Signature moves to steal

- **The interruption ledger.** Every notification type classified: interrupt, badge, or silent log. Defaults biased two levels quieter than the product team's instinct.
- **The year-two mockup.** Design the expert's daily screen explicitly, not just onboarding; ship paths from one to the other.
- **Graceful summon and dismiss.** The product appears fully ready in under 200ms from its global hotkey and vanishes without residue.

## Never copy

- Screenless-ness as a badge of vision on tasks that are inherently visual and spatial. Attention respect means the right bandwidth, not the least.

## Interrogation questions

What does each surface cost in attention and what does it repay? What does this product look like for the two-year veteran? Which notifications would a reasonable user thank us for, and why do the others exist?
