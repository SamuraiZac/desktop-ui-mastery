# Susan Kare - Meaning at Small Sizes

Designer of the original Macintosh icons and typefaces (1983-1986): the trash can, the paintbrush, the smiling Mac, the command key symbol, Chicago and Geneva. Later NeXT, Windows 3.0 card decks, Facebook gifts. Kare proved that a 32x32 grid of black and white pixels can carry personality, metaphor, and instant recognition. Her craft is the foundation of all interface iconography.

## Core principles, translated to desktop UI

1. **Icons are road signs, not illustrations.** A good symbol is read in a glance at a distance, like traffic signage. Detail is the enemy of recognition. If an icon needs more than a second, it failed.
2. **Metaphor from everyday life.** The trash can worked because everyone owns one. Reach for physical-world metaphors that the whole audience shares, and test the metaphor across cultures before shipping (her command key symbol came from a Swedish campground sign precisely because the Apple logo was too loaded).
3. **Constraint breeds clarity.** The 32x32 monochrome grid forced every pixel to justify itself. Self-impose equivalent constraints today: design icons at 16px first, in a single color, on a fixed grid. If it reads at 16px monochrome, it will read anywhere.
4. **Warmth is a feature.** The smiling Mac at boot made a machine feel friendly at a time when computers were intimidating. A small dose of humanity (one playful icon, one warm empty state) does more for trust than any amount of polish. Dose is the operative word.
5. **Craft the whole set, not the hero.** An icon system lives or dies on consistency: shared stroke weight, shared corner logic, shared optical sizing. One beautiful icon in an inconsistent set is worse than a plain consistent set.

## Signature moves to steal

- **The squint test.** Blur or shrink the interface. Hierarchy, groupings, and key actions should survive. If everything turns to uniform grey mush, the visual weighting is flat.
- **Pixel-grid discipline.** Align icon strokes to the pixel grid at target sizes. At 16px, a 1.5px stroke is a smudge. This is why hand-tuned icon fonts and libraries like Lucide feel sharp and auto-scaled SVGs feel soft.
- **One-glyph vocabulary.** Before designing a new icon, check whether an existing platform-standard glyph already means this (gear, magnifier, plus). Invent only where no shared symbol exists; invented glyphs spend recognition budget.
- **Personality in the periphery.** Put character in low-stakes moments: empty states, loading, success. Keep it out of destructive actions and data displays.

## Never copy

- The 1-bit pixel aesthetic as retro decoration. Nostalgic pixel art in a modern tool is style theft without the reasoning.
- Her specific icons. The trash can is now a platform convention; use conventions as conventions, and apply her method to the symbols that do not exist yet.

## Interrogation questions

Does it read at 16px in one color? Is the metaphor universal or local? Does the set share one drawing logic? Where is the single moment of warmth, and is it out of the way of serious work?
