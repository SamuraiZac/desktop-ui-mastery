# Adam Wiggins - Local-First and Malleable Software

Co-founder of Heroku, then of Ink & Switch, the independent industrial research lab behind the local-first software essay (2019, with Kleppmann, van Hardenberg, McGranaghan) and a body of prototypes on tools for thought; co-creator of Muse, the spatial thinking canvas. Wiggins represents research-grade rethinking of the assumptions under desktop software: where data lives, who owns it, and whether users can reshape their tools.

## Core principles, translated to desktop UI

1. **Local-first: the user's device is the primary copy.** The seven ideals from the essay: instant response, multi-device sync, network optionality, real-time collaboration, longevity, privacy, and user ownership. The interface consequences are direct: no spinners on your own data, full offline capability presented as normal rather than degraded, sync as background reconciliation with visible but calm state. Software with these properties feels like property, not tenancy.
2. **Sync is a design surface, not plumbing.** Conflicts, presence, divergence, and merge deserve designed representations: what changed while you were away, whose edit won and why, how to branch and rejoin. Hiding sync until it fails is how users lose work and trust.
3. **Malleable software.** Ink & Switch's research program: users should be able to reshape their tools (rearrange, extend, script, recombine) the way owners modify a workshop. Gradations for shipped products: layout persistence, user-defined views and fields, formula layers, plugin surfaces.
4. **Spatial thinking is unfinished business.** Muse's bet: freeform spatial arrangement (cards on an infinite surface, position carrying meaning) serves ideation in ways lists and documents cannot. For desktop tools with a thinking phase, offer a spatial mode where the geometry belongs to the user.
5. **Research prototypes with honest write-ups.** Ink & Switch publishes what failed alongside what worked. The lab-notebook discipline transfers: every design exploration recorded with its negative results, so the team's taste compounds.

## Signature moves to steal

- **Optimistic-local everything.** Every action commits locally and instantly; the network is an eventual audience, and the UI never blocks on it for the user's own data.
- **The sync ledger.** A calm, inspectable account of what synced, what conflicts arose, and how they resolved, one click from the status indicator.
- **User-owned geometry.** Canvases and boards where the system never auto-tidies the user's spatial arrangement; position is data.

## Never copy

- Local-first as a marketing checkbox atop cloud-dependent architecture; the ideals are testable, and users notice.
- Muse's gestural, stylus-centric interaction model on cursor-driven desktop; steal the spatial philosophy, re-derive the input grammar.

## Interrogation questions

Does any interaction on the user's own data wait on a network? What does the user see when sync conflicts, and is it designed? Which parts of this tool can the user reshape and keep reshaped?
