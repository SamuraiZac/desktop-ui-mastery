# Jordan Singer - Tools That Design

Designer-founder: built a string of playful shipped experiments, then Diagram, the AI design-tools company (Magician, Genius, Automator concepts for Figma) acquired by Figma in 2023, where he led AI product design. Singer is the library's clearest voice on AI-native design tools and on prolific, playful shipping as a design method.

## Core principles, translated to desktop UI

1. **Prompt in, artifact out.** Singer's prototypes established the grammar of AI design assistance: natural language produces an editable, inspectable artifact inside the user's existing tool, never a dead image or a paragraph. Any AI feature in a desktop app should return objects the user can immediately manipulate with the app's normal verbs.
2. **AI as collaborator inside the canvas, not oracle in a sidebar.** The assistance appears where the work is (completing the layout in place, suggesting the next component contextually) rather than in a detached chat pane the user must ferry results from.
3. **Ship toys to find tools.** His method is volume: dozens of small, delightful, public experiments, of which a few reveal a real product. Translation for design practice: prototype features as tiny standalone toys; the ones people play with unprompted are the roadmap.
4. **Lower the floor without lowering the ceiling.** AI features should make the first draft trivial while leaving full manual control untouched; generation that locks users out of editing trades empowerment for dependence.
5. **Craft signals trust in AI features.** Because generative output is unpredictable, the surrounding interface must be exceptionally predictable: clear provenance (what was generated, from what), visible diffs, one-keystroke undo of anything the machine did.

## Signature moves to steal

- **Generate-then-refine loops.** Every generation immediately editable, regenerable in place, and diffable against the previous state.
- **Contextual suggestion ghosts.** Proposed content rendered as accept-or-dismiss ghosts in position, never as modal interruptions.
- **The toy cadence.** A weekly shippable micro-prototype as a team discipline; playfulness as R&D.

## Never copy

- Chat-window-bolted-to-the-side as the AI integration pattern; his entire body of work argues against it.
- Novelty volume without the pruning; the public sees his hits, but the method depends on killing most experiments.

## Interrogation questions

Does every AI output arrive as an editable object with the app's normal verbs? Where is the assistance relative to the work: inside it or beside it? What did the machine change, and can the user see and undo exactly that?
