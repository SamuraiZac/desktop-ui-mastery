# Ryo Lu - Tools for Thought, AI-Native

Head of design at Cursor after five years as an early designer at Notion, with Stripe and Asana before that, placing him at the design helm of two defining products of consecutive eras: the block-based tool for thought and the AI-native code editor. Lu's lens: software as a malleable medium the user shapes, and interfaces where a human and a model work in the same document without the human losing authorship.

## Core principles, translated to desktop UI

1. **Blocks over pages.** Notion's core bet: content decomposed into typed, composable, rearrangeable blocks turns a document tool into a construction kit. Desktop translation: represent the product's substance as manipulable units with shared verbs (drag, transform, nest, reference) rather than as fixed screens with bespoke forms.
2. **The tool is unfinished on purpose.** Notion ships primitives; users build the CRM, the wiki, the tracker. Design the kit and the defaults, then get out of the way, accepting that the best uses will be ones the designer never drew. (The Kay end-user-programming lineage, shipped at mass scale.)
3. **AI belongs in the document, under the cursor.** Cursor's defining pattern: the model proposes changes as inline, diffable, accept-or-reject edits inside the artifact, in flow, rather than as chat output to be transplanted. Any AI-assisted desktop tool should adopt this grammar: proposals in place, provenance visible, authorship retained by the human.
4. **Speed of thought is the target latency.** Tools for thought die at the moment the interface interrupts cognition: a slow block insert, a laggy completion. The interaction budget is the user's train of thought, stricter than any millisecond spec.
5. **Opinionated defaults inside infinite flexibility.** Both Notion and Cursor pair radical malleability with strong out-of-box paths. Flexibility without curated defaults is a blank-canvas paralysis machine.

## Signature moves to steal

- **The universal block handle.** Every content unit exposes the same grip: drag to move, slash or menu to transform, consistent selection and nesting rules product-wide.
- **Inline AI diffs.** Machine suggestions rendered as in-place proposed edits with accept, reject, and refine, never as detached output.
- **Template-as-onboarding.** First-run offers living, editable examples of what the kit can build, teaching the primitives by inhabiting them.

## Never copy

- Notion's specific block chrome and slash-menu anatomy, or Cursor's exact diff presentation; both are current-era signatures under heavy imitation.
- Malleability for products whose value is opinionation. A wallet or a compliance tool should not be a construction kit; choose the Saarinen pole deliberately instead.

## Interrogation questions

What is this product's block: the unit users move, transform, and compose? Where do AI proposals land, and does the human visibly retain authorship? What curated default rescues the user from the blank canvas?
