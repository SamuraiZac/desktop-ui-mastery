# Dieter Rams - Reduction as Discipline

Industrial designer, Braun (1961-1995), author of the Ten Principles of Good Design. The origin point of the reduction lineage that runs through Ive, Saarinen, and every product that calls itself "minimal." Rams never designed software, which is exactly why his principles transfer cleanly: they are about the relationship between a person and a tool, not about pixels.

## Core principles, translated to desktop UI

1. **Good design is as little design as possible.** Not empty; essential. The famous phrase is "Weniger, aber besser": less, but better. The "but better" clause is the part everyone forgets. Deletion is only half the job; what remains must be improved to carry the load.
2. **Good design is honest.** The UI must not promise capability it lacks. A disabled button that looks enabled, a progress bar that lies, a "smart" feature that is a template: all dishonest design.
3. **Good design is unobtrusive.** Tools are neutral and restrained, leaving room for the user's content and self-expression. The user's data is the protagonist. Chrome is the supporting cast.
4. **Good design makes a product understandable.** The structure explains itself. If the interface needs a tour, the structure failed.
5. **Good design is long-lasting.** Avoid the aesthetic of the current quarter. Ask: will this visual decision embarrass us in five years? Gradient meshes and glass blur usually will. A well-set type hierarchy will not.
6. **Good design is thorough down to the last detail.** Nothing is arbitrary. Every radius, every spacing value, every label exists for a stated reason. "Care and accuracy in the design process show respect towards the user."

## Signature moves to steal

- **The single accent.** Braun products were grey, white, black, plus one functional color (the famous yellow or green power switch). Desktop translation: a near-monochrome interface where the one saturated color is reserved exclusively for the primary action or live state. If color appears in twelve places, it appears nowhere.
- **Controls sized by importance, not by symmetry.** The volume knob on a Braun radio is the biggest because it is used the most. Translation: your 100x-a-day action gets the strongest affordance; everything else recedes.
- **Alignment as explanation.** Rams used strict grids so that position alone communicated grouping and hierarchy. Before adding a divider, border, or card, try solving it with alignment and spacing only.
- **The deletion pass.** After a design "works," remove elements one at a time until something breaks, then restore only that. Run this pass explicitly.

## Never copy

- The Braun aesthetic itself (beige, chrome dials, skeuomorphic knobs). Stealing the look while ignoring the reasoning is the exact failure mode this skill exists to prevent.
- Minimalism as style. Empty screens with hidden functionality violate principle 4 (understandable). Rams reduced to essentials, not below them.

## Interrogation questions

When applying Rams, ask of every element: What does this do? What happens if I delete it? Is it honest? Would it still be right in five years?
